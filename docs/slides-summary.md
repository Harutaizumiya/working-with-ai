# Working with AI 内容总结

这份 slides 现在按一条更清晰的主线组织：

变化 → 如何使用 → 方法 → 能力 → 实践 → 扩展 → 认知升级 → 收尾

核心目标不是堆术语，而是逐步建立一个判断：AI 已经不只是代码补全工具，而是需要被设计、被协作、被约束、也值得被信任的工作伙伴。

## 第一阶段：变化

### 1. 标题页 `Working with AI`

开场先定调：

- 这是一套个人实践，不是标准答案
- AI 变化太快，所有经验都只是阶段性的
- 真正重要的是持续磨合自己与 AI 的协作方式

### 2. `Tab → Agent`

这一页解释 AI 编程工具的演进：

- 从单点补全的 `Tab`
- 到多插入点补全的 `Tab Tab`
- 再到用户说目标、AI 自己推进的 `Agent`

重点是：人类输入的粒度越来越高，AI 负责的执行链路越来越长。

### 3. `Agent / Prompt / Context`

这一页拆解三件关键事：

- `Prompt` 决定方向
- `Context` 决定判断质量
- `Agent` 决定事情能不能持续推进直到完成

核心判断是：模型表现并不只由模型本身决定，`Prompt + Context + Agent` 的组合才是体验稳定性的来源。

### 4. `AI 编程的自动驾驶等级`

这里把 AI 编程工具映射成从 `L0` 到 `L5` 的自动驾驶等级。

它传递的重点是：

- `L1 / L2` 还是辅助式工具
- `L3` 开始是 agent 主导、人类 review
- 真正的瓶颈从“会不会写”转向“敢不敢信”

## 第二阶段：问题

### 5. `Research 双模式`

先给出“如何使用 AI”的第一层答案：Research 有两种模式。

- `Context-free`：不带预设地调研，基于第一性原理
- `Context-rich`：结合当前项目结构和限制落地

两者结合，才能既打开思考空间，又不脱离现实约束。

### 6. `一些个人 Prompt 经验`

这部分总结了作者自己的使用原则：

- 用第一性原理思考
- 从根源解决问题
- 不要默认既有思路正确
- 先理解代码再动手
- 相信 AI 的基础能力

这页的作用是把“如何和 AI 说话”提前讲清楚，避免后面 workflow 里出现抽象跳跃。

## 第三阶段：方法论

### 7. `什么是 Skills?`

这页把 `Skill` 定义成给 agent 的“本地说明书”：

- 用自然语言沉淀目标、步骤、判断标准和输出要求
- 解决的是“这类任务应该怎么做”
- 不是工具本身，而是方法论注入

它的价值在于：

- 低成本沉淀团队经验
- 不需要重新训练模型
- 不需要每次重写长 prompt

## 第四阶段：能力与工具

### 8. `我的工具选择`

这一页明确分工：

- `Codex` 负责写代码
- `Gemini` 负责创意类工作
- `ChatGPT` 负责 research
- `Playwright / MCP` 这类工具作为能力接口补足执行环境

它的作用是把后面 workflow 里的工具角色先讲清楚。

## 第五阶段：实践落地

### 9. `我的 Workflow`

这是整场分享的核心页，展示从想法到交付的完整链路：

1. 发现需求
2. GPT 聊天写计划
3. Stitch 绘制 demo 图
4. Google AI Studio 输出原始代码
5. Codex 初始化项目
6. 上下文中注入计划书
7. 根据实际效果不断迭代
8. 最终审核代码提交

这页强调：

- 前半段是快速成型想法
- 后半段是用 Codex 和人类一起把 demo 收敛成可交付代码

也就是说，AI 的价值不只是“生成一段代码”，而是贯穿从想法、原型、脚手架到真实业务落地的整条链路。

## 第六阶段：扩展应用

### 10. `不只是 Vibe Coding`

章节封面，负责把讨论从“写代码”扩展到“协作与沟通”。

### 11. `设计交互协作`

强调开发者和设计师可以围绕 AI 快速生成的原型共同讨论，而不是只靠抽象文字描述。

### 12. `辅助代码理解`

强调 AI 可以帮人快速理解陌生项目、总结结构、标记重点，降低阅读成本。

### 13. `用 AI 做设计`

这一页讨论“AI 味”：

- AI 生成设计常常不是做得差
- 而是平均化、缺少取舍、缺少个性

即使 Skill 可以部分去掉这种平均化，也可能产生新的“模板味”。

## 第七阶段：认知升级

### 14. `一些变化`

章节封面，把讨论从“怎么用 AI”转向“长期使用后的习惯变化”。

### 15. `信息获取方式在变`

观点是：

- 过去靠构造 query
- 现在更像直接表达需求

重点不是单次搜索速度，而是获取答案的整体链路更短。

### 16. `默认不信任“快”`

作者给出一个很有代表性的感受：

- AI 太快反而让人不放心
- “慢”意味着它可能真的在 plan、research、验证

这其实是把人类世界对“可信度”的直觉迁移到了 AI 使用中。

### 17. `AI 在生活中`

这页把 Coding Agent 扩展成通用 Agent：

- 整理相册
- 刷新简历
- 生成赛历
- 制作 slides

核心判断是：只要问题能被描述、拆解、执行、验证，Coding Agent 的工作方式就可以外溢到更多数字场景。

## 第八阶段：收尾

### 18. `Emoji 猜影视作品`

通过一个小游戏把整体氛围从讲述切到互动。

### 19. `Q&A`

最后回到开放讨论，并补充这套 slides 本身也是 AI 协作产物。

## 总结

重排后的结构把整套内容压成一条更连贯的叙事线：

- 先建立共识：AI 正在从补全进入 agent 阶段
- 再回答“怎么用”：Research 和 Prompt
- 然后补方法论：Skill
- 再讲工具分工
- 把高潮放在真实 workflow
- 再扩展到设计、沟通和代码理解
- 最后上升到长期使用后的认知变化

一句话总结：

> AI 不只是更快的补全器，而是一整套需要目标、上下文、方法、工具和信任共同支撑的协作系统。
